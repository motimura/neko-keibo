# 🐱 猫計簿 (Neko-Keibo)

猫にかかる費用や消耗品の在庫を管理するモバイルアプリ。
iOS / Android 両対応。ローカルファーストで端末内にデータを保存し、オフラインで完全に動作します。

## 機能一覧

### 支出管理
- カテゴリ別に猫関連の支出を記録・編集・削除
- 月別の支出一覧表示（スワイプで月切替）
- カレンダーピッカーで日付選択（未来日は選択不可）
- FAB（＋）ボタンからモーダルで支出登録

### ダッシュボード（ホーム画面）
- 月間合計金額の表示
- カテゴリ別円グラフ
- 前月比（金額差・パーセンテージ）
- 在庫切れアラート表示（🔴 X件の在庫が切れそうです）
- スワイプで月切替（未来月は不可）

### 在庫管理
- フード・消耗品は支出登録時に自動で在庫連動
- ステータス自動判定（🟢十分 / 🟡そろそろ / 🔴切れそう）
- 手持ちの在庫を手動登録（「手持ちを登録」機能）
- 在庫詳細モーダルで通知周期の確認・変更
- ステータス悪化時にローカル通知で自動アラート

### 在庫ステータス判定アルゴリズム
- 紐づく最新支出の `reminder_days` を基準に計算
- **残り日数** = 最終購入日 + reminder_days − 今日
- 🟢 **十分**: 残り8日以上、または通知未設定
- 🟡 **そろそろ**: 残り1〜7日
- 🔴 **切れそう**: 残り0日以下
- アプリ起動時・在庫画面フォーカス時に自動更新

### 買い替えリマインダー（通知）
- 支出登録時に通知周期を任意設定（○日後 / ○週間後 / ○ヶ月後）
- 手持ち登録時にも通知設定可能
- 指定期間経過後にローカル通知が届く
  - タイトル: 「🐱 猫計簿」
  - 本文: 「🍗 ロイヤルカナン インドア 2kg そろそろ買い替え時です」
- 通知一覧画面でアクション選択：
  - **同じものを購入** → 同じカテゴリ・品名・金額で自動登録 + 次回通知を再スケジュール
  - **あとで** → 未対応のまま残す
  - **通知をオフ** → このアイテムの定期通知を解除
- 支出一覧から既存の通知設定を変更可能

### データエクスポート / インポート
- **JSONバックアップ**: 支出・在庫・通知の全データを含む完全バックアップ
- **CSVエクスポート**: 支出のみ（家計簿アプリとの互換性を意識、UTF-8 BOM付き）
- **JSONインポート**: 上書き（全削除→INSERT）/ マージ（IDベースで重複スキップ）の2モード
- **CSVインポート**: カテゴリの日本語ラベルからコードに自動変換
- OS共有シートで他アプリと連携

### 設定
- データ管理（エクスポート / インポート / バックアップ復元）
- 全データ削除（確認ダイアログ2回）
- アプリ情報（バージョン、支出/在庫/通知件数）

```bash
npm install
npx expo start
```

## 開発コマンド

```bash
npx expo start                    # 開発サーバー起動
npx expo start --dev-client       # dev-clientで起動
npx expo run:ios --device         # USB接続のiOS実機にビルド＆インストール
npx expo start --ios              # iOSシミュレーター
npx expo start --android          # Androidエミュレーター
npm run test                      # テスト実行
npm run lint                      # Lint
eas build --platform ios          # iOS ビルド（EAS）
eas build --platform android      # Android ビルド（EAS）
```

## データベース

SQLite をローカルDBとして使用。バックエンドサーバー不要。

### テーブル構成

- **expenses**: 支出レコード（カテゴリ、品名、金額、日付、メモ、在庫連動ID、通知設定）
- **inventory**: 在庫アイテム（品名、カテゴリ、ステータス、最終購入日）
- **notifications**: 通知履歴（支出ID、ステータス: pending/purchased/dismissed）

## 制約事項

- iOSのローカル通知スケジュール上限は64件
- 上限を超える場合はアプリ起動時に直近のものから優先的に再スケジュール
- アプリアンインストールで通知は消えるため、再インストール時にSQLiteの履歴から再スケジュール
- アプリアイコン変更は `npx expo run:ios --device` でネイティブ再ビルドが必要（dev-clientのJS更新では反映されない）
